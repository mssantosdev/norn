## Goal

Define the minimal config fields that materially improve v0.0.1 behavior without turning config into a large separate project.

## User Story

As a user, I want Norn to respect a small set of meaningful preferences such as language and theme while remaining easy to understand.

## Acceptance

- preferred language is defined
- theme is defined
- verbosity is defined
- default planning surface is defined
- AI response language and drafting preferences are defined

## Strands

- identify the v0.0.1 fields
- separate stable defaults from deferred advanced config
- align fields with CLI and OpenCode usage
